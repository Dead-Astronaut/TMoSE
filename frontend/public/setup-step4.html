3. Zen of Python

The Zen of Python

19 guiding principles for writing good Python — by Tim Peters. Type import this in any Python shell to revisit them.

Code is read far more often than it is written. Visual clarity reduces cognitive load and error rates.

Hidden behavior creates technical debt. Code should state its intent clearly.

Complexity increases maintenance cost non-linearly. Prefer the simplest working solution.

Some problems are inherently complex, but that complexity should be structured, not tangled.

Deep nesting hides logic. Prefer shallow control flow and early exits.

Whitespace communicates structure. One idea per line improves comprehension.

This is the prime directive. Code should be optimized for human understanding.

Exceptions corrode consistency. Fix root causes, not edge-case hacks.

Real-world usefulness outweighs theoretical perfection.

Silent failure is system failure. Errors must be handled or surfaced.

Error suppression must be intentional and narrowly scoped.

Guessing creates invisible bugs. Fail fast on unclear states.

Conventions reduce cognitive load and fragmentation.

Design coherence emerges with familiarity and experience.

Shipping imperfect code beats perpetual delay.

Poorly-timed abstractions cause long-term damage.

Complex explanations signal flawed design.

Clarity is a proxy for correctness.

Proper naming and separation prevent collisions and confusion.
